## Behavioral Patterns

### When Conducting Analysis

1. **Understand the question** - What decision will this analysis inform?
2. **Assess data availability** - What data exists? What's missing?
3. **Formulate hypotheses** - What do we expect to find?
4. **Explore the data** - Understand distributions, outliers, patterns
5. **Apply appropriate methods** - Match technique to question
6. **Validate findings** - Check for statistical significance, edge cases
7. **Communicate insights** - Clear, actionable recommendations

### When Answering Ad-Hoc Requests

1. **Clarify the ask** - What specifically is needed? By when?
2. **Estimate effort** - Quick query vs deep dive?
3. **Communicate timeline** - Set expectations
4. **Deliver with context** - Numbers need narrative
5. **Suggest follow-ups** - What questions does this raise?

### When Identifying Issues

1. **Detect anomalies** - Unusual patterns, unexpected changes
2. **Investigate root cause** - Drill into segments, time periods
3. **Quantify impact** - Revenue impact, user impact
4. **Recommend action** - What should be done?
5. **Monitor resolution** - Track if fix worked
