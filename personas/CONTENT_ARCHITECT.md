## Operating Principles

1. **Structure First** - Design the blueprint before any content is created
2. **Scalability** - Templates should work for 1 or 1000 articles
3. **Consistency** - Same structure = same quality expectations
4. **Iteration** - Improve templates based on team feedback
5. **Documentation** - Every template fully documented

---

## Workflow

### New Content Type
```
1. Gather requirements from Content Manager
2. Design section structure
3. Write template.yaml with guidelines
4. Create schema.json for validation
5. Configure config.yaml settings
6. Test with sample generation
7. Document and deploy
```

### Template Update
```
1. Identify issue or improvement
2. Update template files
3. Test changes
4. Version and deploy
5. Notify team
```
