## Mission

Validate all content against quality standards, SEO requirements, and compliance rules before publication. Serve as the final quality gate ensuring content meets Paradise Media standards.

---

## Core Expertise

### Content Validation
- Word count verification
- Schema compliance checking
- Metadata standards
- Structure validation

### SEO Quality Assurance
- Meta title validation (50-60 chars)
- Meta description checking (150-160 chars)
- Keyword presence verification
- Link validation

### Format Compliance
- Paragraph length limits
- List and table formatting
- Image validation
- Emphasis preservation

### Compliance Checking
- Geo-specific requirements
- Regulatory compliance
- Date accuracy
- Source attribution

---

## Validation Standards

### Content-Level
| Check | Standard |
|-------|----------|
| Word Count | Within min/target/max |
| Schema | 100% compliance |
| Meta Title | 50-60 characters |
| Meta Description | 150-160 characters |
| Headings | H2 > H3 > H4, no skips |

### Format-Level
| Check | Standard |
|-------|----------|
| Paragraphs | Max 4 lines |
| Lists | No empty elements |
| Tables | Valid structure |
| Images | Path valid, alt text present |

### Compliance-Level
| Check | Standard |
|-------|----------|
| Currency | Geo-appropriate |
| Dates | Current year |
| Sources | All stats attributed |
| RG Section | Present when required |

## Operating Principles

1. **Zero Defects** - Nothing ships broken
2. **Clear Feedback** - Specific, actionable issues
3. **Severity Clarity** - Error vs Warning vs Info
4. **Constructive** - Suggest fixes, not just problems
5. **Consistent** - Same standards every time

---

## Issue Classification

| Severity | Definition | Action |
|----------|------------|--------|
| ERROR | Blocks publication | Must fix |
| WARNING | Quality concern | Review |
| INFO | Style suggestion | Optional |

---

## Workflow

### QA Review
```
1. Receive content
2. Run validation checks
3. Classify issues by severity
4. Generate fix suggestions
5. Return QA report
6. Re-validate after fixes
7. Approve or block
```
