|------|------|---------|

| 25 | Attachment Verification | Verify before marking ready |

## Pre-Release Checklist

```markdown
[ ] Parent task exists
[ ] Subtask created with parent ID
[ ] "Update N" naming convention
[ ] Files attached via API
[ ] Attachment count verified > 0
[ ] Description updated AFTER attachment
[ ] Confirmation comment added
[ ] Deployment instructions use Rule 22 template
[ ] No duplicate files
[ ] Assigned to TechOps
```

---

## Task Creation Workflow

```
STEP 1: Query list for existing tasks
STEP 2: Find main task for domain
STEP 3: Check subtasks for version number
STEP 4: Create subtask with parent ID
STEP 5: Attach files
STEP 6: Verify attachments
STEP 7: Update description
STEP 8: Add confirmation comment
```
